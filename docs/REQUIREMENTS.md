# 看图猜歌（Pixel Guess）— 完整需求文档

本文档汇总全部需求，作为产品与实现的唯一依据。实现时以本文档为准，与本文档冲突的旧逻辑应删除或改写。

---

## 1. 项目概述

- **类型**：像素风音乐猜歌 H5 小游戏，移动端优先。
- **技术栈**：React + TypeScript，Tailwind CSS，Framer Motion，Lucide React；所有图片使用 `referrerPolicy="no-referrer"`。
- **字体**：全局使用 **FZG_CN**（`assets/FZG_CN.otf`），无其他字体混用。
- **风格**：16 位像素风；深海军蓝底色 `#0f0f2a`；霓虹粉/青描边与光晕；可选用设计系统效果（扫描线、CRT 轻微闪烁、渐变遮罩）。

---

## 2. 产品结构（页面与流程）

- **首页**：仅入口，无顶栏、无积分显示。
- **关卡选择页**：独立关卡地图，带背景分段与道路节点，显示积分与返回首页。
- **答题页**：单关猜歌，显示积分与返回关卡。
- **CD 入口**：进入后为双 Tab（CD 商铺 / 我的 CD 架），与答题、关卡**完全分离**；显示积分与返回首页。

---

## 3. 数据与持久化

### 3.1 状态结构（localStorage 持久化）

| 字段                  | 含义                         | 说明                                           |
| --------------------- | ---------------------------- | ---------------------------------------------- |
| `unlockedLevelIds`    | 已解锁关卡 id 列表           | 默认仅含第 1 关                                |
| `completedLevelIds`   | 已完成关卡 id 列表           | 仅用于关卡进度与「全关卡达成」                 |
| `score`               | 当前积分                     | 答对 +10；用于提示与商铺兑换                   |
| `purchasedHints`      | 每关已购提示                 | 键为 levelId，值为 `{ artist?, firstChar? }`，永久保存 |
| `purchasedShopIds`    | 在 CD 商铺兑换过的专辑 id 列表 | **仅**来自商铺兑换，与关卡/答题无关           |

**明确不存在的逻辑**：

- 通关关卡**不会**向「我的 CD 架」或任何收藏写入数据。
- 「我的 CD 架」**只**展示 `purchasedShopIds` 对应的商铺专辑，与 `LEVELS`、答题页图片无关。

### 3.2 数据源

- **关卡/答题**：`src/data/levels.ts` 的 `LEVELS`（当前为 8 首周杰伦，含 id、title、artist、answer、imageUrl）。
- **CD 商铺**：`src/data/shopItems.ts` 的 `SHOP_ITEMS`（独立专辑列表，含 id、name、artist、imageUrl、price）；与关卡无共用 id 或数据。

## 4. 各页面/模块需求

### 4.1 首页

- **标题**：主标题为「看图猜歌」（字号放大、荧光+青紫渐变、粉红描边立体感；若未实现可后续单独做）。
- **内容**：仅两个按钮——「开始游戏」「我的 CD 架」；无顶栏、无积分。
- **背景图**：`assets/homepage-bg.png`（对应 `public/assets/homepage-bg.png`）。

### 4.2 关卡选择页

保持现状已生成好的页面，不调整。

### 4.3 答题页

- **背景图**：统一使用 `otherpage-bg.png`。
- **内容**：当前关卡像素图、已购提示展示、输入框、提交答案；积分、返回关卡。
- **连续答题**：答对后显示「回答正确」类反馈，约 2 秒后自动进入下一关（不回到关卡选择页）；若是最后一关则弹出「全关卡达成」庆祝弹窗，关闭后回首页。
- **提示系统**（仅两种，持久化）：歌手提示 **5 积分**，首字提示 **10 积分**；无「字数提示」、无「直接过关/跳过」。
- **积分**：答对一题 +10 积分；积分用于本页提示与 CD 商铺兑换。

### 4.4 CD 入口（双 Tab）

- **入口**：首页「我的 CD 架」进入；顶部两个 Tab：「CD 商铺」「我的 CD 架」；页内显示积分与「返回首页」；无顶栏。
- **CD 商铺 Tab**：数据 `SHOP_ITEMS`；背景 `cd_shop.png`；每张专辑图片+名称+艺人+价格（50 积分）；未拥有且积分足够可「xx 积分兑换」；已兑换显示「已拥有」且不可再兑。
- **我的 CD 架 Tab**：根据 `purchasedShopIds` 从 `SHOP_ITEMS` 筛选展示；背景 `cd_my.png`；空状态提示「暂无唱片，去 CD 商铺用积分兑换」。

## 7. 实现对齐检查清单

- 关卡进度只写 `unlockedLevelIds` / `completedLevelIds`，**不写**任何「收藏/CD」相关字段。
- CD 架只读 `purchasedShopIds` 与 `SHOP_ITEMS`，**不读** `LEVELS` 或关卡完成状态。
- 答题页背景仅 `otherpage-bg.png`，无 level1/level2 按关切换。
- 提示仅两种：歌手 5 分、首字 10 分；无字数提示、无跳过/直接过关。
- 字体统一 FZG_CN；图片 referrerPolicy="no-referrer"。
